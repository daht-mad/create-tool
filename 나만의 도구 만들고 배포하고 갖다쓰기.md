# 나만의 도구 만들고 배포하고 갖다쓰기

# 배경

- 클코로 일을 시키다보면 그때그때 엄청난 기능들을 만들어줌. 근데 이때 만든 기능을 재사용하기가 어려운 문제.
- **그러면 우연히?어쩌다? 만들어버리게 된 기능을 모듈화 하여 내가 다른 비슷한 작업할 때도 갖다 쓰게 하면 어떨까? (도구처럼)**
- 더 나아가서 우리 팀원들이 공동 저장소에, 각자 만든 기능들을 모아놓고 갖다 쓰게 하면 어떨까?
- 더더 나아가서 비개발자 누구나 서로 한줄명령어 입력으로 갖다 쓸 수 있으면 좋겠다
- 그러려면 동일한 아키텍쳐로 기능도구를 만들 수 있는 가이드가 있으면 되겠다
- 그러면 더더더 나아가서 비개발자 누구나 도구만드는도구로 뚝딱 만들어서 배포할 수 있게 할 수도?

# 기존 문제의식

- 기존 도구 저장소들이 다 개발자 친화적임. 오픈소스로 만들어놨다고 해도 비개발자는 그걸 어떻게 사용해야하는지 모름.
    - 클코 스킬 저장소 : [https://github.com/anthropics/skills](https://github.com/anthropics/skills)
    - mcp 저장소 : [https://smithery.ai/servers](https://smithery.ai/servers)
    - 기타 등등
- 그리고 그곳에 만들어 배포할 수 있는 건 개발자들 뿐임. (비개발자는 못함)
- 즉, “비개발자 친화적인 편리하고 편안한” 도구저장소가 필요함.
    - 비개발자도 누구나 만들어서 딸깍 배포
    - 비개발자도 누구나 필요한 것 딸깍 설치

### 참고해 볼 유사 서비스

- smithery
    - 리니어, 에어테이블, 구글드라이브, 슬랙 등 자주 쓰는 커넥터를 딸깍으로 연동하게 한 뒤
    - 채팅 UI로 해당 커넥터들을 활용한 워크플로/스킬을 만들 수 있게끔 하는 기능까지 만듦
    - 그러나 결국 스미더리 들어와서 쓰고, 그걸 다른 곳(ex. 클로드코드 또는 로컬폴더)에 가져가서 써야함.
    - **핵심 : CLI 에이전트(ex. 클로드코드, 안티그래비티, 커서 등)에서 만들고, 배포하고, 사용하는 모든 것이 가능해야 함. 즉, 플랫폼은 커뮤니티 기능만 남을 것 (서로 공유 및 기여할 수 있고, 기여에 대한 보상을 받을 수 있는)**

# 그래서 내가 만든 것

## **비개발자도 AI와 대화만으로 자동화 도구를 만들고 배포할 수 있는 시스템**

Claude Code의 슬래시 커맨드 기능을 활용하여:

- 자연어로 원하는 기능을 설명하면 Claude가 코드 생성, 테스트, 배포까지 자동 처리
- 만들어진 도구는 curl 한 줄로 누구나 설치 가능
- 만드는 사람, 사용하는 사람 모두 Git, npm, 터미널 명령어를 몰라도 사용 가능

---

### 1. 비개발자도 누구나 만들어서 딸깍 배포

```
나: /create-tool
Claude: 어떤 기능을 만들고 싶으신가요?
나: 이미지를 압축해서 파일 크기를 줄이고 싶어요

[3분 후]

Claude: ✅ img-compress 도구 생성 완료!
설치 명령어: curl -o .claude/commands/img-compress.md ...
```

### 2. 비개발자도 누구나 필요한 것 딸깍 설치

```bash
# 설치 (복사-붙여넣기 한 번)
mkdir -p .claude/commands && curl -o .claude/commands/md2pdf.md <https://raw.githubusercontent.com/daht-mad/md2pdf/master/.claude/commands/md2pdf.md>

# 사용
/md2pdf README.md
```

    - 즉, 어쩌다 만들어버린 기능을 나만의 도구로 만들어 배포하기

# 시대적 흐름 - 비개발자의 개발 인프라 구축

- **필요한 걸 말로 만들고 필요한 걸 말로 시킨다**
    - 코드 먼저가 아닌 대화 먼저!
    - smithery.ai에 “채팅으로 스킬 만드는” 흐름이 추가된 건, MCP 서버/스킬을 코드 먼저가 아니라 대화형으로 설계·생성할 수 있게 해주는 기능이라고 보면 된다.
- **학습 자체는 대부분 AI와 대화하며 커스텀 방식으로 하게될 것**
    - 즉, 별도 앱을 만드는 방향보다는 AI에게 구조와 지침을 주고 이 인프라를 활용한 교육으로 가는 방향
    - 학습자 커스텀 가이드문서, 가이드영상을 제작하는게 아니라 애초에 에이전트(AI)를 “시켜서 하게끔” 하는 방식으로 접근하기.
    - 즉, 전제는 “사람이 한다“가 아니라, “에이전트 시킨다“가 될 거 같음
    - **AI가 도와주지 못하는 지점을 어떻게 사람들끼리 서로 돕게끔 만들 것인가가 핵심**
        - 혹은 사람들끼리만 서로 가능한 부분이 무엇인지 (ex. 팀코칭? 휴먼터치? 서로돕기?)
- **남이 만든 걸 쓰기보다 점점 내가 만들고 소유하고 싶어질 것**
    - **왜?**
        - 기본적으로 이제는 누가 만든 거 보면 → 나도 이거 클코한테 이거 따라서 만들어달라고 하면 될 거 같은데? 그걸 왜 돈주고 써?(비용이슈) 내껄로 만들어서 100% 커스터마이징 해서 쓰고 싶은데?(커스텀니즈)
    - **그렇다면?**
        - **남이 내껄 어떻게 쓰게 할 것인가가 아니라 - 남이 어떻게 쉽게 내껄 베껴가게 할 것인가, 그리고 그 나의 기여를 어떻게 자산화 할 것인가**
        - 만든 기능, 도구, 서비스 사용자가 거의 나 개인 또는 우리 팀 수준임
        - 그러면 트래픽이나 api 리밋, 속도 고려 안해도 될 것 같음 (이런 걸 고려하기 시작하면 사실 비개발자가 클로드코드 써도 완벽 배포는 불가함) → 비개발자 친화적임
    - **그래서 만들면 다 오픈소스로 푸는 게 핵심일 것 같고, 해당 서비스를 통해 수익화를 하는 게 아니라 “기여”로 인한 수익화가 가능해지면 좋겠음**
        - 누구나 서비스 생산자가 되는 시대 → 점점 소비자보다 생산자가 많아지게 된다 → 공급이 많아지고 수요가 줄어지는 형국 → 공급으로 돈을 버는 게 아닌 다른 방식으로 돈을 버는 구조가 필요하다
    - **고민의 방향**
        - 어떻게 하면 사람들이 개인이 만든 것을 오픈소스로 기꺼이 공유하게끔 할 것인가 : 기여의 자산화, 수익화 방향성
            - 오픈소스로 많이 배포/기여할 수록 ‘나’라는 사람을 증명할 수 있게 됨 → 퍼스널 브랜딩, 협업 연결, 1인사업가 전환 → 시대적 흐름과 결이 맞으며, 시대적 흐름을 촉진해줄 것
        - 세상의 90%는 비개발자임. 비개발자가 쉽게 공유하고, 쉽게 가져다 쓸 수 있는 인프라 구축 필수적
    - 오픈소스 예시 - 커서맛피아님
        - 커맨드, 스킬, 에이전트 만드는 프롬프트 오픈소스로 공개하심 : [https://github.com/greatSumini/cc-system/tree/main](https://github.com/greatSumini/cc-system/tree/main)
        - [monet.design](http://monet.design) 이라는 웹사이트 섹션별 컴포넌트 추출, 공유 플랫폼을 만들었는데 누구나 기여 가능하도록 함 → 오픈소스 장점이 사람들이 알아서 수정해준다고 함

# 향후 계획

- 자동화시스템 때 만든 어드민페이지 처럼, 이미 만들어진 도구들을 모아두고 찾아보고 설치할 수 있는 한 줄 명령어 볼 수 있는 플랫폼을 만들어야겠다 (약간 스미더리 같은?)
    - 클로드코드용, 안티그래비티용 탭 나눠서 복붙할 한줄명령어 확인 가능
- 마이페이지를 만들어서 ‘내가 만든 도구들’ 모아볼 수 있게 하고,
- 더 나아가 팀페이지 만들어서 ‘우리 팀이 만들고 함께 공유할 도구들’ 모아볼 수 있게 하고
    - 비개발자 조직에서는 어쩌다 만들어버린 프로젝트, 기능들을 어떻게 같이 쓸지조차 모름
    - 다오랩만 해도 각자 다오랩을 위한 기여로 이것저것 프로젝트/기능을 만들고 있는데 한 곳에 모여져있지 않음
        - 다오챗 / 공지사항 모인 대시보드 / 기여 측정 대시보드 / 모임 앨범,기록 아카이브 플랫폼 등
    - 핵심은 비개발자 팀/조직/커뮤니티 구성원 누구나 쉽게 만들고, 오픈소스로 배포하고, 함께 쓸 수 있는 생태계 구축
- **오픈소스로 하되, 서로의 기여를 측정할 수 있는 시스템을 만들기 (블록체인 기술 기반)**
    - 내가 만들어 배포하면 몇 점
    - 남이 내꺼 좋아요 누르면 몇 점
    - 남이 내꺼 클론해서 사용하면 몇 점
    - 남이 내꺼 클론해서 사용하면서 기능 개선하면 다시 해당 도구 하위에 배포 가능 → 다단계 형태
        - 최초 기여자 > 클론 기여자 > 클론클론 기여자 …
        - 암웨이 다단계 방식을 생각해보자 (다단계로 데려온 사람이 또 다른 사람 데려오면 최초 데려온 사람에게도 일정 포션이 배분됨)
