# 출력 패턴

일관된 고품질 출력을 위한 가이드입니다.

## 템플릿 패턴

구조화된 출력이 필요할 때 사용합니다.

### 엄격한 템플릿

API 응답, 설정 파일 등 정확한 형식이 필요한 경우:

```json
{
  "name": "@[스코프]/[도구이름]",
  "version": "1.0.0",
  "bin": {
    "[스코프토큰]-[도구이름]": "./bin/[도구이름].js"
  }
}
```

### 유연한 템플릿

상황에 맞게 조정이 필요한 경우:

```markdown
# [도구이름]

[기능 설명 - 상황에 맞게 작성]

## 설치

[설치 방법 - 환경에 따라 조정]

## 사용법

[사용 예시 - 도구 특성에 맞게 작성]
```

## 예시 패턴

구체적인 입출력 예시로 기대 결과를 명확히 합니다.

### 커밋 메시지 예시

**입력:**
```
새 파일 추가하고 버그 수정함
```

**출력:**
```
feat: 새 기능 추가 및 버그 수정

- 새 파일 추가
- 기존 버그 수정

🤖 Generated with Claude Code
```

### 완료 메시지 예시

**입력:**
도구 생성 완료 상태

**출력:**
```
🎉 도구 생성 완료!

📦 저장소: https://github.com/username/tool-name

📥 팀원 설치 명령어:
mkdir -p .claude/commands && curl -o .claude/commands/tool-name.md \
  https://raw.githubusercontent.com/username/tool-name/master/.claude/commands/tool-name.md

🚀 사용법:
- Claude Code: /tool-name [인자]
- 터미널: username-tool-name [인자]
```

## 진행 상황 표시 패턴

사용자에게 진행 상황을 알립니다.

```
✓ 프로젝트 구조 생성 완료
✓ TypeScript 설정 완료
✓ 코드 작성 완료
⏳ 빌드 중...
✓ 빌드 완료
✓ 테스트 통과
⏳ GitHub에 푸시 중...
✓ 배포 완료!
```

## 에러 메시지 패턴

문제 상황을 명확히 전달합니다.

```
❌ 에러: 파일을 찾을 수 없습니다

원인: 지정된 경로에 파일이 존재하지 않습니다.
경로: /path/to/file.md

해결 방법:
1. 파일 경로가 올바른지 확인하세요
2. 파일이 존재하는지 확인하세요
3. 상대 경로 대신 절대 경로를 사용해보세요
```

## 핵심 원칙

> 예시는 설명보다 스타일과 세부 수준을 더 명확하게 전달합니다.

- 추상적 설명보다 구체적 예시 사용
- 일관된 이모지와 기호 사용
- 다음 단계를 명확히 안내
